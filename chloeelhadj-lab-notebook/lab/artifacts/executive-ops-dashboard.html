Key Decisions

  • Removed the AR aging module after repeated reliability issues. A smaller dashboard built on trustworthy, validated data is more useful than a larger one built on uncertain data.
  • Reads directly from the source system instead of cached outputs to ensure every run reflects current data rather than relying on cached pipeline outputs, for freshness on every run.
  • Resolve a data-binding issue so the live dashboard renders correctly instead of showing blank values.
  • Kept the sales module intentionally limited until line-item data becomes available.
